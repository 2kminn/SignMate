# SignMate 제품 요구사항 정의서(PRD)

## 1. 제품 개요

### 1.1 제품명

SignMate

### 1.2 한 줄 설명

카메라 기반 손동작 인식을 활용해 기초 수어를 해석하고 학습할 수 있는 모바일 웹
애플리케이션이다.

### 1.3 제품 목적

- 수어 입문자가 일상에서 자주 사용하는 기초 수어를 쉽게 학습하도록 돕는다.
- 카메라를 이용해 자신의 동작을 확인하고 즉시 피드백을 받게 한다.
- 반복 학습과 퀴즈를 통해 학습 성취도를 확인하게 한다.

### 1.4 현재 개발 단계

프론트엔드 MVP 단계다.

- React 기반 모바일 우선 UI 구현 완료
- 실제 브라우저 카메라 연결 완료
- 수어 인식 결과는 mock 데이터 사용
- 학습 및 퀴즈 기록은 브라우저 `localStorage` 사용
- MediaPipe 또는 별도 인식 모델 연동 예정
- 회원, 서버 API, 데이터베이스는 미구현

## 2. 배경 및 해결할 문제

수어 입문자는 정적인 이미지나 설명만으로 정확한 손 모양과 움직임을 익히기 어렵다.
또한 자신이 수행한 동작이 올바른지 바로 확인하기 어렵다.

SignMate는 다음 문제를 해결한다.

| 문제 | 해결 방향 |
|---|---|
| 기초 수어를 어디서부터 배워야 할지 어렵다 | 카테고리별 기초 표현 제공 |
| 텍스트와 이미지만으로 동작을 이해하기 어렵다 | 카메라 기반 따라 하기 제공 |
| 자신의 동작이 맞는지 확인하기 어렵다 | 인식 결과와 일치도 피드백 제공 |
| 학습 성과를 확인하기 어렵다 | 퀴즈 점수와 최근 기록 제공 |

## 3. 목표 및 비목표

### 3.1 MVP 목표

- 사용자가 별도 가입 없이 바로 서비스를 체험할 수 있다.
- 최소 5개의 기초 수어를 탐색하고 학습할 수 있다.
- 카메라로 수어 해석, 따라 하기, 퀴즈를 수행할 수 있다.
- 퀴즈 결과와 학습 기록을 같은 브라우저에서 다시 확인할 수 있다.
- 모바일 화면에서 핵심 기능을 불편 없이 사용할 수 있다.

### 3.2 MVP 비목표

- 모든 한국수어 문장 번역
- 음성 또는 자연어를 수어 영상으로 변환
- 사용자 간 소셜 기능
- 강사 및 관리자용 학습 관리 시스템
- 여러 기기 간 기록 동기화
- 의료, 행정 등 전문 통역 대체

## 4. 주요 사용자

### 4.1 수어 입문자

- 수어를 처음 배우거나 기초 표현을 복습하려는 사용자
- 간단하고 짧은 학습 경험을 선호한다.
- 자신의 동작에 대한 즉각적인 피드백이 필요하다.

### 4.2 수어에 관심 있는 일반 사용자

- 특정 수어의 의미나 표현 방법을 빠르게 확인하려는 사용자
- 회원가입 없이 기능을 체험하고 싶어 한다.

## 5. 핵심 사용자 시나리오

### 5.1 첫 실행

1. 사용자가 SignMate에 처음 접속한다.
2. 온보딩에서 서비스의 주요 기능을 확인한다.
3. 시작 버튼을 선택한다.
4. 수어 해석 탭으로 이동한다.
5. 이후 같은 브라우저에서는 온보딩을 자동으로 생략한다.

### 5.2 수어 해석

1. 사용자가 수어 해석 탭에서 카메라 기능을 시작한다.
2. 브라우저의 카메라 권한을 허용한다.
3. 카메라 앞에서 수어를 표현한다.
4. 시스템이 인식된 수어 이름과 일치도를 표시한다.
5. 사용자가 세션을 종료한다.

### 5.3 수어 학습 및 연습

1. 사용자가 학습 탭으로 이동한다.
2. 이름을 검색하거나 카테고리를 선택한다.
3. 수어 카드를 선택해 설명, 예문, 동작 가이드를 확인한다.
4. 따라 하기 버튼을 선택한다.
5. 카메라 앞에서 목표 수어를 표현한다.
6. 시스템이 목표 수어와의 일치 여부를 표시한다.

### 5.4 퀴즈

1. 사용자가 퀴즈 탭에서 퀴즈를 시작한다.
2. 화면에 제시된 단어를 카메라 앞에서 수어로 표현한다.
3. 시스템이 정답 여부를 표시한다.
4. 사용자가 다시 시도하거나 다음 문제로 이동한다.
5. 모든 문제 완료 후 정답 수와 점수를 확인한다.
6. 점수는 브라우저에 저장되어 퀴즈 탭에 표시된다.

## 7. 데이터 요구사항

상세 구조와 관계는 [ERD](./ERD.md)를 따른다.

### 7.1 MVP 데이터

- 수어 마스터 데이터
- 온보딩 완료 여부
- 학습을 시작한 수어 목록
- 퀴즈 점수 및 저장 시각

### 7.2 기본 데이터 규칙

- 수어는 변경되지 않는 영문 `label`로 식별한다.
- 인식 신뢰도는 `0.0` 이상 `1.0` 이하로 관리한다.
- 점수는 `0` 이상 `100` 이하의 정수로 저장한다.
- 브라우저 기록은 JSON 파싱 실패 시 기본값으로 복구한다.
- 카메라 원본 영상은 사용자의 명시적 동의 없이 저장하지 않는다.

## 8. 비기능 요구사항

### 8.1 사용성

- 주요 버튼의 최소 높이는 44px 이상으로 한다.
- 모바일 세로 화면을 우선 지원한다.
- 카메라 권한 요청 전 사용 목적을 사용자가 이해할 수 있어야 한다.
- 오류 발생 시 기술 용어보다 사용자가 취할 행동을 안내한다.

### 8.2 접근성

- 아이콘만 있는 버튼에 접근 가능한 이름을 제공한다.
- 상태 변화와 인식 피드백은 색상만으로 구분하지 않는다.
- 동적 결과 영역은 스크린 리더가 인지할 수 있도록 구성한다.
- 텍스트와 배경은 WCAG AA 수준의 명암비를 목표로 한다.

### 8.3 성능

- 일반적인 모바일 네트워크에서 초기 화면을 3초 이내 표시하는 것을 목표로 한다.
- 카메라 프리뷰 진입 후 2초 이내 사용 가능한 상태를 목표로 한다.
- 실시간 인식은 사용자 체감 지연 300ms 이하를 목표로 한다.
- 모델 파일은 지연 로딩하고 재방문 시 캐시를 활용한다.

### 8.4 호환성

- 최신 Chrome, Safari, Edge 모바일 브라우저를 우선 지원한다.
- 카메라 기능은 `localhost` 또는 HTTPS 환경에서만 동작한다.
- 카메라 미지원 환경에서는 학습 콘텐츠 열람 기능을 유지한다.

### 8.5 개인정보 및 보안

- MVP에서는 카메라 영상을 서버로 전송하거나 저장하지 않는다.
- 가능한 경우 손동작 추론을 사용자 기기에서 수행한다.
- 향후 서버 전송이 필요하면 사전 동의, 전송 목적, 보관 기간을 명시한다.
- 로그인 도입 시 퀴즈 점수는 서버에서 검증 및 계산한다.

## 9. MVP 수용 기준

다음 조건을 모두 충족하면 MVP 핵심 기능이 동작하는 것으로 판단한다.

### 9.1 공통

- 모바일 화면에서 가로 스크롤 없이 핵심 화면을 사용할 수 있다.
- 해석, 학습, 퀴즈 탭 사이를 이동할 수 있다.
- 새로고침 후에도 온보딩 및 학습 기록이 유지된다.

### 9.2 학습

- 등록된 5개 수어가 모두 표시된다.
- 검색 결과와 카테고리 필터 결과가 정확하다.
- 수어 상세 정보와 따라 하기 화면에 올바른 목표 수어가 표시된다.

### 9.3 카메라

- 권한 허용 시 실제 카메라 영상이 표시된다.
- 권한 거부 시 앱이 중단되지 않고 재시도 가능한 안내를 표시한다.
- 실제 모델 연동 후 인식 결과와 신뢰도가 화면에 갱신된다.

### 9.4 퀴즈

- 각 문제에서 목표와 예측 `label`이 같을 때만 정답 처리된다.
- 마지막 문제 완료 후 정답 수와 환산 점수가 정확하다.
- 완료 결과가 저장되고 퀴즈 탭의 최고 및 최근 점수에 반영된다.

## 10. 성공 지표

실사용 배포 후 다음 지표를 수집한다.

| 지표 | 정의 | 초기 목표 |
|---|---|---:|
| 온보딩 완료율 | 온보딩 시작 사용자 중 완료 비율 | 80% 이상 |
| 학습 진입률 | 방문 사용자 중 수어 상세를 연 비율 | 50% 이상 |
| 연습 시작률 | 상세 열람 사용자 중 카메라 연습 시작 비율 | 40% 이상 |
| 퀴즈 완료율 | 퀴즈 시작 사용자 중 전체 문제 완료 비율 | 60% 이상 |
| 카메라 성공률 | 권한 요청 중 정상 프리뷰 시작 비율 | 90% 이상 |
| 재방문율 | 첫 방문 후 7일 안에 재방문한 사용자 비율 | 측정 후 목표 확정 |

개인정보 보호를 위해 원본 영상 없이 이벤트 단위로 측정하는 것을 원칙으로 한다.
